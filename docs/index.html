rundeck-admin

commands to manage your rundecks

SYNOPSIS

rerun [ARGS] rundeck-admin: [ apitoken-create | apitoken-list | apitoken-remove | project-create | project-export | project-import | project-list | project-replicate | resource-add | resource-get | schedule-takeover | ssh-copy-id ] [OPTIONS]

README

The rundeck-admin commands help you manage your rundecks.

Requirements

  • rerun
  • curl
  • xmlstarlet
  • expect (for ssh-copy-id)

Download

Download a shell archive from bintray.

COMMANDS

  • apitoken-create: Create an API token.
  • apitoken-list: List the API tokens.
  • apitoken-remove: remove an api token
  • project-create: Create a project.
  • project-export: Export a project.
  • project-import: Import a project archive.
  • project-list: List the projects.
  • project-replicate: Replicate the project(s) from one rundeck to another.
  • resource-add: add a resource to a project resource model.
  • resource-get: get resource info.
  • schedule-takeover: Claim all scheduled jobs from another cluster server.
  • ssh-copy-id: Copy an SSH key to another server.

OPTIONS

  • archive: the archive file.
  • file: file to store result.
  • format: the file format.
  • hostname: the server hostname.
  • keypath: path to ssh key file.
  • model: the resource model.
  • password: the login password.
  • project: the project name.
  • resource: the resource name.
  • to-url: target rundeck instance.
  • token: the login token.
  • url: rundeck server url.
  • user: the login user name.
  • uuid: the cluster server uuid.

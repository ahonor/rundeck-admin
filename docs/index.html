rundeck-admin

commands to manage your rundecks

SYNOPSIS

rerun [ARGS] rundeck-admin: [ apitoken-create | apitoken-list | project-create | project-export | project-import | project-list | project-replicate | ssh-copy-id ] [OPTIONS]

README

The rundeck-admin commands help you manage rundeck deployments.

Use the apitoken-* commands to create and list API tokens.

Use the project-* commands to create, list, export, import and replicate projects.

Requirements

  • bash and a posix environment
  • curl
  • xmlstarlet

Notes

  • This module should probably be renamed given its widening scope.

COMMANDS

  • apitoken-create: Create an API token
  • apitoken-list: List the API tokens.
  • project-create: create a project
  • project-export: export a project
  • project-import: import a project archive
  • project-list: list the projects
  • project-replicate: replicate the project(s) from one rundeck to another.
  • ssh-copy-id: Copy an SSH key to another server.

OPTIONS

  • archive: the archive file
  • hostname: the server hostname
  • keypath: the path to the key file.
  • password: the login password
  • project: the project name
  • to-url: target rundeck instance
  • url: rundeck server url
  • user: the login user name

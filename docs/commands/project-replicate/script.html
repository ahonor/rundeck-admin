Error handling

This script is designed to fail-fast.

Trap errors and exit. The call to rerun_die will print the the error message and exit with the error command exit status. 


trap 'rerun_die $? "*** command failed: rundeck-archive:project-replicate. ***"' ERR

Run [set] nounset to treat unset variables as errors. Set [pipefail] so a pipeline return status is the value of the last (rightmost) command to exit with non-zero status.


set -o nounset -o pipefail

Command implementation

Put the command implementation here.

The credentials used to login to the rundecks

CREDS="--user ${USER} --password ${PASSWORD}"

List the existing projects in the source rundeck.

from_list=( $(rerun rundeck-archive:project-list $CREDS --url $URL) )
target_list=( $(rerun rundeck-archive:project-list $CREDS --url $TO_URL) )

Match them against the name or bash regexp.

projects=( )
for p in ${from_list[*]}
do
    [[ $p =~ $PROJECT ]] && projects=( ${projects[*]:-} $p )
done

if [ ${#projects[*]} -lt 1 ]
then
    echo >&2 "Project \"$PROJECT\" did not match any in the list: [$list]"
    exit 0
fi

echo >&2 "Replicating projects: [${projects[*]:-}] ..."

for project in ${projects[*]:-}
do
     rerun rundeck-archive:project-export $CREDS --url $URL \
        --project ${project} --archive $project.zip

Check if the project exists in the target

     if ! rerun_list_contains $project ${target_list[*]} 
     then

Create the project in the target.

         rerun rundeck-archive:project-create $CREDS \
             --url $TO_URL --project $project
     fi

     rerun rundeck-archive:project-import $CREDS --url $TO_URL \
        --project ${project} --archive $project.zip 
done

Done. Exit with last command exit status.

exit $?
